# Мой Зал — нативное приложение для iPhone (SwiftUI)

Нативная версия сделана специально ради двух возможностей, которых **не может**
дать веб‑приложение на iOS:

1. **Уведомление об окончании отдыха приходит на Apple Watch, даже когда
   приложение полностью закрыто.** Таймер отдыха планирует системное локальное
   уведомление (`UNTimeIntervalNotificationTrigger`) — iOS доставит его в срок,
   даже если приложение выгружено из памяти, и автоматически продублирует на
   часы, когда iPhone заблокирован. Уровень `.timeSensitive` пробивает режим
   «Не беспокоить».
2. **Данные из «Здоровья» берутся автоматически.** Через HealthKit приложение
   само читает тренировки с Apple Watch (пульс средний/максимальный, калории,
   дистанция) и вес тела, а `HKObserverQuery` с фоновой доставкой обновляет их,
   как только появляются новые — без ручного экспорта файлов.

> Веб‑версия (в корне репозитория) продолжает работать как раньше — это
> отдельный продукт. Нативная нужна только для тех, кому важны пункты выше.

## Что нужно для сборки

- **Mac** с установленным **Xcode 15+** (бесплатно в App Store).
- Аккаунт Apple — для запуска на своём iPhone хватит **бесплатного** Apple ID
  (приложение придётся переустанавливать раз в 7 дней). Для установки без
  ограничений и фоновой доставки Health — платный **Apple Developer Program**
  (~$99/год).
- iPhone с iOS 16.4+ и сопряжённые Apple Watch.

## Сборка — вариант A (рекомендуемый, через XcodeGen)

```bash
brew install xcodegen        # если ещё не установлен
cd ios
xcodegen generate            # создаст MoyZal.xcodeproj из project.yml
open MoyZal.xcodeproj
```

В Xcode:
1. Выбери таргет **MoyZal → Signing & Capabilities**.
2. В **Team** выбери свой Apple ID (Add an Account…, если нужно).
3. Проверь, что в **Capabilities** есть **HealthKit** (добавляется автоматически
   из entitlements; если нет — нажми «+ Capability» → HealthKit).
4. Подключи iPhone кабелем, выбери его в списке устройств вверху и нажми ▶ (Run).
5. На iPhone: Настройки → Основные → Управление устройством → доверять твоему
   сертификату разработчика (только при первом запуске с бесплатным Apple ID).

## Сборка — вариант B (без XcodeGen, вручную)

1. Xcode → File → New → Project → **iOS App**, интерфейс **SwiftUI**, язык
   **Swift**, имя **MoyZal**.
2. Удали автоматически созданные `ContentView.swift` и `MoyZalApp.swift`.
3. Перетащи в проект все файлы из папки `MoyZal/Sources/` (Copy items if needed).
4. Таргет → Signing & Capabilities → **+ Capability → HealthKit**.
5. Открой Info.plist и добавь два ключа с текстом (можно взять из `project.yml`):
   - **Privacy - Health Share Usage Description** (`NSHealthShareUsageDescription`)
   - **Privacy - Health Update Usage Description** (`NSHealthUpdateUsageDescription`)
6. Собери и запусти на iPhone (▶).

## Как проверить обе возможности

**Уведомление на часах при закрытом приложении:**
1. Запусти приложение, разреши уведомления при первом запросе.
2. Начни тренировку, добавь упражнение, отметь подход — запустится отдых.
3. Заблокируй iPhone (или сделай свайп‑вверх и закрой приложение) и опусти руку.
4. По истечении отдыха уведомление «Отдых окончен» придёт на Apple Watch.
   (Длительность отдыха по умолчанию 90 сек; можно уменьшить в коде
   `store.restSeconds` для быстрой проверки.)

**Автоматические данные Здоровья:**
1. Вкладка **Здоровье** → «Разрешить доступ к Здоровью» → включи все категории.
2. Тренировки с Apple Watch, пульс, калории и вес появятся автоматически;
   вкладка **История** покажет их вперемешку с тренировками в зале.

Сейчас нативная версия покрывает ядро: тренировка + таймер отдыха с
уведомлениями + автоматический HealthKit + история. Программы, рецепты,
калькуляторы и ИИ‑тренер пока только в веб‑версии — их можно перенести
в нативную позже, если решишь полностью уйти на неё.
